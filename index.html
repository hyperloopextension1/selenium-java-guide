Java
BasePage.java
public class BasePage {                  // Class
    private WebDriver driver;             // Encapsulation — private field
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {   // Constructor
        this.driver = driver;
        this.wait   = new WebDriverWait(driver, Duration.ofSeconds(15));
        PageFactory.initElements(driver, this);
    }

    public WebDriver getDriver() { return driver; }   // getter

    protected void click(WebElement el) {
        wait.until(ExpectedConditions.elementToBeClickable(el)).click();
    }
    protected void type(WebElement el, String text) {
        el.clear(); el.sendKeys(text);
    }
    protected String getText(WebElement el) {
        return wait.until(ExpectedConditions.visibilityOf(el)).getText();
    }
}

public class LoginPage extends BasePage {  // Inheritance
    @FindBy(id="username") private WebElement usernameField;
    @FindBy(id="password") private WebElement passwordField;
    @FindBy(id="submit")   private WebElement submitBtn;

    public LoginPage(WebDriver driver) { super(driver); }

    public DashboardPage login(String u, String p) {
        type(usernameField, u);
        type(passwordField, p);
        click(submitBtn);
        return new DashboardPage(getDriver());
    }
}

// Runtime Polymorphism (Method Overriding)
public class DriverFactory {
    public static WebDriver getDriver(String browser) {
        WebDriver driver;
        switch(browser.toLowerCase()) {
            case "chrome":
                WebDriverManager.chromedriver().setup();
                driver = new ChromeDriver();   // WebDriver = ChromeDriver (polymorphism)
                break;
            case "firefox":
                WebDriverManager.firefoxdriver().setup();
                driver = new FirefoxDriver();
                break;
            default:
                WebDriverManager.edgedriver().setup();
                driver = new EdgeDriver();
        }
        driver.manage().window().maximize();
        return driver;
    }
}

🛡️ Try-Catch-Finally in Automation

Java
public boolean isElementPresent(By locator) {
    try {
        driver.findElement(locator);
        return true;
    } catch (NoSuchElementException e) {
        return false;
    }
}

// Multiple catch blocks
public void clickElement(By locator) {
    try {
        driver.findElement(locator).click();
    } catch (NoSuchElementException e) {
        System.err.println("Element not found: " + locator);
        throw new RuntimeException("Click failed — element not found", e);
    } catch (ElementNotInteractableException e) {
        System.err.println("Element not interactable, trying JS click");
        ((JavascriptExecutor) driver).executeScript("arguments[0].click();",
            driver.findElement(locator));
    } finally {
        // always runs — use for cleanup
        Logger.info("clickElement executed for: " + locator);
    }
}

try (FileInputStream fis = new FileInputStream("data.xlsx");

public interface BrowserActions {
    void openBrowser(String url);
    void closeBrowser();
    default void takeScreenshot(String name) {  // default method (Java 8)
        // implementation here
    }
    static String getVersion() { return "4.0"; }  // static method (Java 8)
}

// Abstract class — common base for all page objects
public abstract class BasePage implements BrowserActions {
    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(15));
    }

    public abstract void verifyPageLoaded();  // each page must implement

    protected void waitAndClick(WebElement el) {   // shared concrete method
        wait.until(ExpectedConditions.elementToBeClickable(el)).click();
    }
}

// Concrete page
public class LoginPage extends BasePage {
    public LoginPage(WebDriver d) { super(d); }

    @Override
    public void verifyPageLoaded() {
        wait.until(ExpectedConditions.titleContains("Login"));
    }
    @Override public void openBrowser(String url) { driver.get(url); }
    @Override public void closeBrowser() { driver.quit(); }
}

XPath vs CSS — When to use what

CSS: Prefer for speed and readability. Works great for most element finding.
XPath: Use when you need text() matching, parent traversal (..), sibling axes (following-sibling), or dynamic index-based selection. CSS cannot traverse UP the DOM — XPath can.

Selenium Waits

⛔ Never Use

Thread.sleep(3000) — always waits full duration, makes tests slow and unreliable. Always use Explicit Wait.

⚠️ Don't Mix

Never mix Implicit + Explicit waits in the same test — it causes unpredictable behavior (double-timeout bug in some browsers).

Implicit vs Explicit vs Fluent

public class DriverFactory {
    // ThreadLocal for parallel test execution (one driver per thread)
    private static final ThreadLocal<WebDriver> driver = new ThreadLocal<>();

    public static WebDriver initDriver(String browser) {
        WebDriver d;
        switch(browser.toLowerCase()) {
            case "chrome":
                WebDriverManager.chromedriver().setup();
                ChromeOptions co = new ChromeOptions();
                co.addArguments("--start-maximized", "--disable-notifications");
                d = new ChromeDriver(co);
                break;
            case "firefox":
                WebDriverManager.firefoxdriver().setup();
                d = new FirefoxDriver();
                break;
            case "headless":
                WebDriverManager.chromedriver().setup();
                ChromeOptions hco = new ChromeOptions();
                hco.addArguments("--headless=new", "--window-size=1920,1080");
                d = new ChromeDriver(hco);
                break;
            default: throw new IllegalArgumentException("Unsupported: "+browser);
        }
        d.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
        driver.set(d);
        return d;
    }

    public static WebDriver getDriver() { return driver.get(); }

    public static void quitDriver() {
        if(driver.get() != null) {
            driver.get().quit();
            driver.remove();  // prevent memory leak in thread pool
        }
    }
}

public class RetryAnalyzer implements IRetryAnalyzer {
    private int count = 0;

    }

public class ExcelUtils {

    public static Object[][] readData(String path, String sheet) {
        try(FileInputStream fis = new FileInputStream(path);
            Workbook wb = new XSSFWorkbook(fis)) {

            Sheet s = wb.getSheet(sheet);
            int rows = s.getLastRowNum();     // excludes header
            int cols = s.getRow(0).getLastCellNum();
            Object[][] data = new Object[rows][cols];

            for(int r=1; r<=rows; r++) {
                for(int c=0; c<cols; c++) {
                    Cell cell = s.getRow(r).getCell(c);
                    data[r-1][c] = getCellValue(cell);
                }
            }
            return data;
        } catch(IOException e) { throw new RuntimeException(e); }
    }

    private static Object getCellValue(Cell cell) {
        if(cell == null) return "";
        switch(cell.getCellType()) {
            case STRING:  return cell.getStringCellValue();
            case NUMERIC: return cell.getNumericCellValue();
            case BOOLEAN: return cell.getBooleanCellValue();
            default: return "";
        }
    }
}

🏗️ Framework Design

Page Object Model (POM): Design pattern. Each page = one class. Locators defined using By objects. Elements found using driver.findElement(By.id("...")) at the time of use (fresh lookup every call).

PageFactory: Implementation of POM. Uses @FindBy annotations + PageFactory.initElements(). Elements are lazy-initialized (found on first use). Supports @CacheLookup to cache element after first find.

// POM style By usernameBy = By.id("username"); driver.findElement(usernameBy).sendKeys("admin"); // PageFactory style @FindBy(id = "username") WebElement usernameField; PageFactory.initElements(driver, this); // in constructor usernameField.sendKeys("admin");

Caution: @CacheLookup can cause StaleElementReferenceException on dynamic pages.
BaseTest: Parent class for all test classes. Contains @BeforeMethod (init driver, open URL) and @AfterMethod (quit driver, capture screenshot on failure). Ensures every test starts with clean state.

BasePage: Parent class for all Page classes. Contains shared methods: click(), type(), waitForElement(), isDisplayed(), getText(), scrollTo(). Test code stays clean — page methods handle WebDriver interactions.

// BaseTest public class BaseTest { @BeforeMethod public void setUp() { DriverFactory.initDriver(ConfigReader.get("browser")); DriverFactory.getDriver().get(ConfigReader.get("url")); } @AfterMethod public void tearDown(ITestResult result) { if(result.getStatus() == ITestResult.FAILURE) captureScreenshot(result.getName()); DriverFactory.quitDriver(); } }
In testng.xml: parallel="methods" thread-count="4". WebDriver is stored in ThreadLocal<WebDriver> in DriverFactory — ensures each thread gets its own browser instance. ExtentReports tests stored in ThreadLocal<ExtentTest> for thread-safe report logging. Remove ThreadLocal in @AfterMethod to prevent memory leaks.

🧪 TestNG Questions

TestNG annotations control test lifecycle. Execution order:

@BeforeSuite → runs once before all suites @BeforeTest → runs before <test> tag in testng.xml @BeforeClass → runs once before all methods in a class @BeforeMethod → runs before EACH test method @Test → the actual test @AfterMethod → runs after EACH test method @AfterClass → runs once after all methods in a class @AfterTest → runs after <test> tag @AfterSuite → runs once after all suites

Key attributes: @Test(priority=1, groups="smoke", enabled=true, timeOut=5000, retryAnalyzer=RetryAnalyzer.class, dependsOnMethods="loginTest")
@BeforeClass: Runs once before all test methods in the class. Use for expensive one-time setup (launching browser, DB connection).
@BeforeMethod: Runs before EACH test method. Use for per-test setup (navigate to URL, login, clear state).

In parallel testing: @BeforeMethod is safer because each thread gets its own setup/teardown cycle.
Hard Assert (org.testng.Assert): Stops test execution immediately on first failure. Use when subsequent steps depend on this assertion passing.

Soft Assert (org.testng.asserts.SoftAssert): Collects all failures, continues execution. Must call softAssert.assertAll() at the end — reports all failures. Use when you want to verify multiple independent conditions in one test.

SoftAssert sa = new SoftAssert(); sa.assertEquals(actualTitle, "Dashboard"); sa.assertTrue(logoutBtn.isDisplayed()); sa.assertAll(); // reports all failures at once
@DataProvider supplies test data to @Test methods. The method returns Object[][] — outer array = rows (test runs), inner array = columns (parameters).

@DataProvider(name = "loginData") public Object[][] getData() { return new Object[][] { {"admin@test.com", "Admin@123", true}, {"user@test.com", "User@123", true}, {"wrong@test.com", "wrong", false} }; } @Test(dataProvider = "loginData") public void loginTest(String email, String password, boolean shouldPass) { loginPage.login(email, password); if(shouldPass) Assert.assertTrue(dashboard.isLoaded()); else Assert.assertTrue(loginPage.getError().isDisplayed()); }

For Excel: return ExcelUtils.readData("testdata.xlsx", "Login").
Groups let you categorize tests and run specific subsets.

@Test(groups = {"smoke", "regression"}) public void loginTest() { ... } @Test(groups = {"regression"}) public void cartTest() { ... }

Run only smoke in testng.xml:
<groups> <run> <include name="smoke"/> </run> </groups>

Run from Maven: mvn test -Dgroups=smoke

Best practice: Define groups — smoke (critical path ~5 min), regression (full suite), sanity (post-deploy check).
@Test(dependsOnMethods = "loginTest") ensures loginTest runs first. If loginTest FAILS, the dependent test is SKIPPED (not failed).

@Test public void loginTest() { ... } // must pass @Test(dependsOnMethods = "loginTest") public void addToCartTest() { ... } // skipped if login fails @Test(dependsOnMethods = "loginTest", alwaysRun = true) public void logoutTest() { ... } // runs EVEN if login fails

⚠️ Use sparingly — creates test coupling. Prefer independent tests. Use for true sequential flows (login → checkout → payment).
Create ExcelUtils class using Apache POI. readData(filePath, sheetName) opens workbook, iterates rows/columns (skip row 0 as header), returns Object[][]. Create @DataProvider that calls ExcelUtils.readData(). Annotate @Test with dataProvider="name". TestNG calls the test once per row, passing column values as parameters. Use parallel=true on @DataProvider for faster execution.
Configure in testng.xml:

<suite name="Suite" parallel="methods" thread-count="4">

Parallel modes:
• methods: Each @Test method runs in separate thread. Most granular.
• classes: Each test class runs in separate thread.
• tests: Each <test> tag runs in separate thread.
• instances: Each instance of a class runs in separate thread.

Required for parallel safety:
1. ThreadLocal<WebDriver> in DriverFactory
2. ThreadLocal<ExtentTest> in TestListener
3. No shared mutable static state
4. @BeforeMethod and @AfterMethod (not @BeforeClass)
5. Each test creates its own test data

🥒 BDD & Cucumber Questions

TDD (Test-Driven Development): Write failing unit test → write code to pass it → refactor. Developer-focused. Tests are code.

BDD (Behavior-Driven Development): Write behavior scenarios in plain English (Gherkin) → write step definitions → automate. Collaboration between devs, QA, business. Tests are readable by non-technical stakeholders.

BDD in automation: Cucumber reads .feature files (Gherkin) → maps to step definition methods → executes Selenium/API code.

Given: precondition/context
When: action/event
Then: expected outcome
And/But: additional steps
Gherkin is a plain-English language for writing BDD scenarios. Keywords: Feature, Scenario, Given, When, Then, And, But, Background, Scenario Outline, Examples.
